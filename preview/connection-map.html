Section 2
Page Navigation
Which pages navigate to which other pages, what triggers the navigation, and how it works. All navigation goes through onNavigate(viewId) — a callback passed from App.jsx to every view. No view imports another view directly.
How navigation works: App.jsx owns activeView state. It passes onNavigate = (id) => setActiveView(id) to every view as a prop. A view calls onNavigate("journal") to switch — App.jsx updates activeView and the ViewRouter renders the new component. No React Router. No URLs. Pure state.
Who navigates where — and why

→LandingPage — not logged in
→Auth — sign in / sign up flow
→Onboarding — first session (tb_onboarded not set)
→App — logged in + onboarded

→onNavigate("journal") — "Most recent entry" action
→onNavigate("journal") — "Write now" empty state CTA
→onNavigate("contradictions") — "Open questions / Resolve →"
→onNavigate("contradictions") — "Active contradictions / Full view →"
ThoughtBiography.jsx
→onNavigate("biography") — "Generate biography →" button (unlocks at 10 entries)
ThoughtGraph.jsx
→onNavigate("drift") — "View drift →" when user clicks a concept node
Reminders.jsx

Section 3
API Calls Per Page
Every HTTP request each frontend component makes, the method, the endpoint, and what it does with the response.
App.jsx — shell-level calls
GET /dashboard on auth + on every view switch loads stats (entries, concepts, contradictions) for sidebar counters
GET /health every 30 seconds drives the green/red connection dot in the sidebar
Auth.jsx
POST /auth/register on register submit creates account → saves tokens → calls onAuthenticated(user)
POST /auth/login on login submit validates credentials → saves tokens → calls onAuthenticated(user)
POST /auth/refresh auto on 401 silently refreshes expired access token using stored refresh token
Dashboard.jsx
GET /dashboard on mount stats + recent entries + open questions + daily prompt + active contradictions
ThoughtBiography.jsx — Journal Editor
GET /entries?limit=50 on mount loads entry list for right panel
POST /entries on save (Cmd+Enter) saves entry → triggers Claude extraction → returns entry + concepts
DELETE /entries/{id} on delete confirm removes entry and its concept links from graph
POST /transcribe on voice recording stop sends audio blob → gets transcript → inserts into textarea
ThoughtGraph.jsx — D3 Force Graph
GET /graph on mount returns {nodes, edges} → D3 builds force simulation
GET /concepts/{label} on node click loads concept detail panel: history, connections, drift, entries
ContradictionWorkflow.jsx
GET /graph on mount reads CONTRADICTS edges only → builds radial map and list
ConceptDriftTimeline.jsx
GET /concepts?limit=50 on mount populates concept picker list in left panel
GET /concepts/{label}/drift on concept select time-series of stability + sentiment → feeds Recharts LineChart
BiographyDocument.jsx
GET /biography on mount eligibility check — shows entry count, entries needed, can_generate flag
GET /biography/history on mount loads history sidebar (previous generations)
POST /biography/generate on "Generate" click SSE stream — progress events per pass → final biography text → auto-saved
POST /biography/save manual save persists current biography text to Neo4j
InfluenceTrees.jsx
GET /influence-trees on mount D3-ready hierarchical data: books/people as roots, concepts as leaves
TemporalPlayback.jsx
GET /temporal-data on mount + on scrub graph state at a date — ?date=2023-06-01 → D3 re-renders
Search.jsx
GET /search?q=...&mode=... on query change (debounced) mode: fulltext | semantic | concept. Returns matching entries.
Reminders.jsx
GET /reminders/prompts on mount (parallel) AI-generated reflection prompts based on graph state
GET /reminders/digest on mount (parallel) digest preview data for current week
POST /reminders/generate on "Refresh" click generates new prompts on demand
PUT /reminders/settings on settings change saves prompt frequency and concept filter preferences
Export.jsx
GET /export?format=json on "Download JSON" all entries + concepts as structured JSON
GET /export?format=markdown on "Download Markdown" all entries as Markdown with concept annotations
GET /export?format=graph on "Download Graph" graph topology as JSON (nodes + edges)
TherapistMode.jsx — Shared Access
GET /shares on mount list of active share links with access logs
POST /shares on "Create link" generates new read-only share token → returns URL

DigestSettings.jsx
GET /digest/settings on mount loads current preferences: day, hour, enabled, last_sent
PUT /digest/settings on save updates day/hour/enabled in Neo4j User node
POST /digest/preview on "Preview" click generates and returns HTML digest — rendered inline

Section 5
Full Data Flow Traces
End-to-end traces for the three most important user actions. Every hop shown.
Writing and saving a journal entry
#LayerFileWhat happens
1FrontendThoughtBiography.jsxUser types. Presses Cmd+Enter. handleSave() fires.
2FrontendAuth.jsx — authFetchauthFetch injects Bearer token into POST /entries headers.
3Backendapi.pyget_current_user() validates JWT → extracts user_id.
4Backendencryption.pyencrypt(content, key_from_password) → AES-256-GCM ciphertext.
5DatabaseNeo4jCREATE (:Entry {id, user_id, content: ciphertext, ...}) CREATE (u)-[:AUTHORED]->(e)
6ExternalAnthropic Claudeclaude-sonnet extracts concepts, influences, summary from plaintext.
7DatabaseNeo4jMERGE (:Concept) ON CREATE/ON MATCH — create or update concept node. CREATE (e)-[:CONTAINS {sentiment, importance, framing}]->(c)
8Backendapi.pyResponse: {id, content (plaintext), concepts: [...], ai_summary}
9FrontendThoughtBiography.jsxPrepend new entry to list. Show concept chips on the card. Increment entry counter.
Generating a biography (SSE stream)
#LayerFileWhat happens
1FrontendBiographyDocument.jsxUser clicks "Generate". POST /biography/generate sent. EventSource opened.
2Backendbiography_api.pyCheck rate limit (12h). Returns 429 if too soon. Otherwise opens SSE StreamingResponse.
3Backendbiography_generator.pypull_graph_data() — queries Neo4j for concepts, contradictions, influences, oldest/newest entries.
4FrontendBiographyDocument.jsxSSE event: {type:"progress", pass:1, label:"Reading your concepts...", pct:10} → progress bar 10%
5ExternalClaude Sonnet (×4)Passes 1-4: structure, arc, contradictions, voice calibration.
6FrontendBiographyDocument.jsxProgress events at each pass. Bar advances 10→25→40→52%
7ExternalClaude OpusPass 5: first full biography draft. 150-220 words.
8ExternalClaude SonnetPass 6: truth check against raw entries. Flags overstated claims.
9ExternalClaude OpusPass 7: final prose. Tighten, elevate, check rhythm.
10Backendbiography_api.pySSE event: {type:"complete", biography:"...", flags:[...]} → save to Neo4j.
11FrontendBiographyDocument.jsxBiography text renders in literary serif. History sidebar updates.
Authentication lifecycle
#LayerFileWhat happens
1FrontendRoot.jsxAuthStorage.isLoggedIn() → false → render Auth.jsx
2FrontendAuth.jsxUser submits login. POST /auth/login.
3Backendauth.pybcrypt.verify(password, hash). If valid: create access_token (24h JWT) + refresh_token (30d JWT).
4FrontendAuth.jsx — AuthStoragesessionStorage.set('tb_access', token). localStorage.set('tb_refresh', refresh). Call onAuthenticated(user).
5FrontendRoot.jsxlocalStorage.getItem('tb_onboarded') ? → App : → Onboarding
6FrontendAuth.jsx — authFetchEvery subsequent request: injects Authorization: Bearer access_token automatically.
7Backendauth.pyOn 401: access_token expired. authFetch auto-calls POST /auth/refresh.
8FrontendAuth.jsx — authFetchNew access_token saved. Original request retried transparently.
9FrontendApp.jsxLogout: AuthStorage.clear() → sessionStorage + localStorage cleared → setAuthed(false) → Auth.jsx shown.
